### 部署至 Vercel

將 Next.js 應用程式部署到生產環境最簡單的方法是使用由 Next.js 開發者們所構建的 [Vercel]() 平台。

Vercel 是一個無伺服器平台，適用於靜態和混合應用程式，可與您的無頭內容、電子商務或數據庫整合。我們致力於幫助前端團隊部屬、預覽和提供令人愉悅的使用者體驗，並以優秀的性能作為預設。您可以免費開始使用 — 無需信用卡。

### 建立 Vercel 帳戶

首先，前往 <a href="https://vercel.com/signup" target="_blank">vercel.com/signup</a> 創建一個 Vercel 帳戶。選擇**使用 GitHub 繼續**並完成註冊過程。

### 導入 `nextjs-blog` 程式碼儲存庫

一旦完成註冊，將您的 nextjs-blog 程式碼存儲庫**導入**到 Vercel 上。您可以從這裡進行導入：<a href="https://vercel.com/import/git" target="_blank">vercel.com/import/git</a>。

- 您需要為 **GitHub 安裝 Vercel**。您可以給予它對**所有存儲庫**的訪問權限。
- 一旦完成安裝，導入 `nextjs-blog`。

您可以使用***預設值***來設置以下項目 — 無需進行任何修改。Vercel 會自動檢測您是否有 Next.js 應用程式，並為您選擇最佳的構建配置。

- 項目名稱
- 根目錄
- 構建命令
- 輸出目錄
- 開發命令

當您進行部署時，您的 Next.js 應用程式將開始進行構建。通常情況下，構建過程應該在一分鐘內完成。

  **可用幫助**：如果您部署失敗，可以隨時在 <a rel="noopener noreferrer" target="_blank" href="https://github.com/vercel/next.js/discussions">GitHub 討論區</a>尋求幫助。要了解更多關於部署的資訊，請參閱<a rel="noopener noreferrer" target="_blank" href="/docs/deployment">我們的文件</a>。

當部署完成後，您將獲得**部署的 URL**。點擊其中一個 URL，您應該可以看到 Next.js 的初始頁面在線上運行。
